## Design Context

### Users
- **주 사용자**: 암호화폐 모의투자에 관심 있는 한국인 투자자 (초보~중급)
- **사용 맥락**: 전략을 백테스트하고, 검증된 전략으로 모의투자 봇을 운용하며, 커뮤니티에서 성과를 공유
- **핵심 Job**: "내 전략이 실제로 돈을 벌 수 있는지 데이터로 검증하고 싶다"
- **관리자**: 실매매 운용 + 플랫폼 관리

### Brand Personality
- **3단어**: 친근 · 심플 · 접근성
- **톤**: 전문적이지만 위압적이지 않음. 투자 용어를 쓰되 초보자도 이해할 수 있는 수준
- **감정 목표**: 확신과 통제감 + 흥미와 탐험 + 안정과 신뢰
- **레퍼런스**: TradingView (차트 중심, 데이터 밀도, 프로 느낌이면서 접근 가능)
- **안티패턴**: 카지노/도박 느낌, 과도한 수익 강조, 복잡하고 위압적인 UI

### Aesthetic Direction
- **테마**: 다크 모드 기본 + 라이트 모드 지원
- **스타일**: 글라스모피즘 패널, 미묘한 배경 그래디언트, rounded-2xl 카드
- **폰트**: Plus Jakarta Sans (제목), Inter (본문)
- **팔레트**:
  - Primary: #3B82F6 (Blue) — 주요 액션, 선택 상태
  - Secondary: #10B981 (Green) — 성공, 수익, 가동 중
  - Accent: #8B5CF6 (Purple) — 그래디언트, 강조
  - Danger: #EF4444 (Red) — 손실, 삭제, 실매매 경고
- **아이콘**: Lucide React (일관된 스트로크 스타일)
- **애니메이션**: ease-out-quart 커브, staggered entrance, 150-500ms 범위
- **접근성**: prefers-reduced-motion 대응, WCAG AA 포커스 링

### Design Principles

1. **데이터가 주인공이다** — 장식보다 정보. 차트, 수치, 상태가 가장 눈에 띄어야 한다.
2. **한 화면에 한 가지 결정** — 정보 과부하 대신 단계별 안내. 복잡성은 점진적으로 공개.
3. **검증이 투자의 시작** — 백테스트 → 모의투자 → (관리자만) 실매매. 이 흐름이 자연스러워야 한다.
4. **실수를 방지하되 막지 않는다** — 위험 고지, 확인 모달, 되돌릴 수 없는 작업 경고. 그러나 사용자의 선택을 존중.
5. **테마 토큰 우선** — 하드코딩 색상 금지. 모든 색상은 CSS 변수 기반 시맨틱 토큰(th-text, th-border 등) 사용.

### Tech Stack
- Next.js 14, TypeScript, Tailwind CSS
- Radix UI (Dialog, Select, Tooltip, Popover, Label)
- class-variance-authority + tailwind-merge (컴포넌트 변형)
- Lucide React (아이콘), Recharts (차트)
- CSS 변수 기반 테마 시스템 (다크/라이트)

### Component Patterns
- `glass-panel` / `glass-panel-hover` — 글라스 카드의 기본 스타일
- `Button` (CVA) — primary, danger, ghost, outline, secondary 변형
- `Badge` — success, warning, danger, info, default 변형
- `ModalWrapper` + `ModalHeader` — Portal 기반 모달
- `EmptyState` — icon + title + description + action(CTA) 구조
- `PageContainer` — 페이지 래퍼 (max-w-7xl, animate-fade-in-up)
- `StatCard` — 통계 카드 (glass-panel + hover gradient)
