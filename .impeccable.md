# Design Constitution: Theme Tokens First

## Core Principle
All UI elements must derive their appearance from the CSS variable-based theme tokens. Hardcoded opacity levels on 'white' or 'black' are technical debt.

## Forbidden Patterns (DO NOT GENERATE)
1. `bg-white/[0.04]` -> USE `bg-th-card-hover`
2. `border-white/[0.06]` -> USE `border-th-border-light`
3. `text-white/80` -> USE `text-th-text-secondary`
4. `placeholder-gray-500` -> USE `placeholder-th-text-muted`

## Quality Gate
Any PR containing a literal hex code `#` or a Tailwind base color `gray-`, `blue-` (except primary) in `frontend/src` must be REJECTED by the Reviewer agent.
